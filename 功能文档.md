# Python后端\+Web前端 炫酷本地音乐播放器 完整版功能需求文档（内嵌歌词\+外置LRC双兼容）

## 一、文档简介

本文档为**Python后端 \+ Web前端 前后端分离架构**音乐播放器最终完整版功能规范，区别于纯前端版本，新增Python服务端支撑，兼顾**本地离线使用\+服务端能力拓展**。核心保留**音频内嵌歌词 \+ 外置LRC歌词双模式全兼容**，不删减任何原有歌词、特效、播放功能，通过Python实现文件解析、元数据读取、歌词预处理、文件托管服务，前端负责界面渲染、特效展示、播放交互，搭载炫酷UI、实时音频频谱、六种全屏歌词动效、完整播放控制与持久化存储，功能全覆盖、逻辑无冲突，可直接作为前后端开发落地标准文档。

**核心规则（最终定稿，完全沿用）**

- 默认优先级：**音频内嵌歌词优先加载**

- 无内嵌歌词时：自动启用外置LRC兜底功能

- 有内嵌歌词时：支持手动上传外置LRC**强制覆盖替换**

- 支持一键切换、解绑、恢复默认歌词来源，双模式自由切换，互不冲突

## 二、产品定位与运行环境

### 2\.1 产品定位

一款基于**Python \+ Web前端**的高颜值、功能闭环的PC端音乐播放器，采用轻量前后端分离架构。Python后端负责文件接收、音频解析、内嵌歌词提取、LRC预处理、本地文件托管；Web前端负责交互操作、炫酷UI渲染、音频播放、全屏歌词动效、音频可视化。支持本地音乐离线播放，同时依托后端解决纯前端文件解析限制、大文件解析卡顿、歌词解析容错低等问题，主打**双歌词兼容机制\+沉浸式全屏炫酷歌词动效\+服务端音频解析**，适配本地无损音乐播放、精准歌词同步、高端视觉特效体验需求。

### 2\.2 运行环境

- 运行方式：本地启动Python服务，浏览器访问本地网页运行，全程本地离线、无公网请求

- 适配设备：PC端电脑（1080P/2K/4K分辨率）

- 适配浏览器：Chrome、Edge、Firefox、Safari 最新主流版本

- 运行特性：所有音频解析、歌词解析、文件处理由Python后端完成，前端专注渲染交互，数据全程本地存储，无外网上传

### 2\.3 架构模式

轻量前后端分离架构：**Python后端（接口服务\+文件解析）\+ 静态Web前端（交互\+特效）**，无复杂框架、无数据库依赖、部署简单、启动即用。

播放器整体分为七大核心模块，新增Python后端服务模块，原有前端功能模块全部保留、适配优化：Python后端服务模块、本地音乐文件管理模块、双歌词解析管理模块、音乐播放控制模块、炫酷视觉特效模块、全屏歌词动效模块、辅助工具与持久化模块。

## 四、技术栈整体方案（新版）

- 服务框架：Flask 轻量Web框架，快速搭建本地API服务、静态资源托管

### 4\.2 Web前端技术栈

- 基础架构：HTML5 \+ CSS3 \+ JavaScript ES6\+

- 音频播放：HTML5 Audio API、AudioContext 音频频谱解析

- 特效渲染：CSS3 动画、Canvas 粒子/星空/律动/流光特效

- 数据交互：Axios 请求后端本地API，获取解析后的音频、歌词数据

- 样式框架：原生CSS自定义炫酷样式，轻量无冗余依赖

## 五、Python后端核心功能

### 5\.1 基础服务能力

- 本地服务启动：默认本地127\.0\.0\.1端口启动，仅本机可访问，安全私密

- 静态资源托管：托管前端HTML/CSS/JS、静态素材，实现一键访问页面

- 文件托管服务：接收前端上传的音频、LRC文件，临时读取解析，不永久存储冗余文件

- 统一数据接口：提供标准化API，返回歌曲元数据、内嵌歌词、解析状态

### 5\.2 音频解析核心接口（解决纯前端短板）

- 批量解析音频文件：读取MP3/FLAC/M4A/WAV/OGG文件，精准提取歌名、歌手、专辑、时长、专辑封面

- 内嵌歌词深度解析：精准读取音频ID3v2标签、无损音乐元数据中的同步Lyrics歌词，修复纯前端解析失败、乱码、格式错乱问题

- 歌词格式标准化：将各类非标内嵌歌词统一转换为标准LRC格式，返回结构化时间轴数据

- 封面编码处理：将音频内嵌封面转为前端可直接展示的Base64图片格式

### 5\.3 LRC文件处理接口

- 外置LRC文件解析：后端精准解析LRC文本、时间轴、双语歌词，容错处理破损、格式不规范的LRC文件

- 歌词绑定管理：记录每首歌曲的「内嵌歌词原始数据」「外置LRC绑定数据」

- 歌词切换接口：接收前端切换指令，返回对应生效歌词数据

### 5\.4 本地数据持久化（后端）

后端通过本地JSON文件持久化核心数据，重启服务不丢失：

- 歌曲列表信息、歌曲唯一标识、歌词来源标记

- 每首歌绑定的外置LRC文本、歌词切换状态、解绑状态

- 播放历史、收藏列表核心数据

### 6\.2 双歌词核心模块（核心重点，前后端协同）

#### 6\.2\.1 歌词加载优先级规则（最终版不变）

1. 歌曲导入后，Python后端优先解析音频**内嵌同步歌词**，返回结构化LRC数据

2. 解析到有效内嵌歌词：前端自动加载并同步播放，默认生效

3. 无内嵌歌词/内嵌歌词解析失败：前端自动启用外置LRC导入功能兜底

4. 支持手动覆盖：无论歌曲是否有内嵌歌词，均可手动上传外置LRC，**强制替换当前歌词**

5. 支持一键恢复：覆盖外置LRC后，可一键解绑，恢复原始内嵌歌词

#### 6\.2\.2 内嵌歌词解析逻辑

- 后端全权解析：处理MP3、FLAC、M4A主流音频的内嵌同步歌词、双语歌词

- 格式标准化：统一转换为标准LRC时间轴格式，前端无需复杂解析，直接渲染

- 容错处理：后端识别标签损坏、歌词格式异常，返回状态标识，前端友好提示，不影响音乐播放

#### 6\.2\.3 外置LRC歌词功能（完整保留，无删减）

- 手动绑定：支持点击上传、拖拽LRC文件至播放器，前端上传至后端完成解析绑定

- 自动匹配：支持同目录同名匹配（歌曲\.mp3 \+ 歌曲\.lrc）自动绑定

- 歌词解析：后端兼容所有标准LRC格式，容错处理特殊格式、破损LRC文件

- 独立管理：支持LRC解绑、重新上传、清空歌词缓存、手动编辑歌词文本

- 兜底能力：无内嵌歌词时，外置LRC为唯一歌词来源，正常使用所有歌词特效

#### 6\.2\.4 歌词切换管理功能

- 前端切换开关：一键切换「使用内嵌歌词 / 使用外置LRC歌词」

- 后端双缓存：持久化存储每首歌的内嵌歌词、外置LRC原文，切换无延迟

#### 6\.2\.5 通用歌词同步逻辑

后端输出统一结构化歌词数据，前端共用一套高精度同步机制：

- 毫秒级进度匹配，跟随音乐播放实时同步
